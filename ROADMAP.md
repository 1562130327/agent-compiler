# Agent Compiler — 路线图

## 项目定位

**LLM 调用去重引擎。** 把语义相似的请求归并到同一个参数化工作流模板上执行，避免对 LLM 的重复调用。与所有 Agent 框架（OpenClaw、Hermes、LangChain 等）是互补关系，不是竞争关系。

## 核心原理

```
第一次:  用户输入 → L1(规则)✗ → L2(缓存)✗ → L3 LLM 推理(4.6s) → 编译为模板 → 缓存
后续相似: 用户输入 → L1(规则)✗ → L2 语义缓存命中(0.4ms) → 直接执行 — 快 10000 倍
高频操作: 用户输入 → L1 规则命中(0ms) → 直接执行 — 无限快
```

## v0.2 — 通用 Agent 加速中间件

**目标：让任何 Agent 框架都能零侵入接入，立即节省 Token 成本。**

### 功能清单

- [ ] **代理认证** — API Key 校验，防止未授权使用
- [ ] **限流保护** — 单用户/单 IP 速率限制，防止上游 API 超额
- [ ] **多租户隔离** — 每个 Agent 实例拥有独立的缓存命名空间，互不污染
- [ ] **缓存预热** — 预定义常见问题集，批量调用 LLM 存入缓存
- [ ] **跨模型缓存** — 切换到不同 LLM 模型时缓存依然有效（纯语义相似度匹配）
- [ ] **统计接口增强** — `/stats` 返回节省的 Token 数、预估费用、各层命中趋势
- [ ] **Web Dashboard** — 实时查看命中率、缓存条目、省了多少 Token
- [ ] **缓存失效策略** — 支持手动清理、TTL 过期、LRU 淘汰
- [ ] **多实例共享缓存** — 多个 Proxy 实例共享 Redis/PostgreSQL 后端

### 应用场景

- 同时运行 OpenClaw + Hermes，两个都接同一个 Proxy
- 团队共用一个 Proxy，A 同事的执行经验 B 同事自动复用
- 本地缓存积累后导出到生产环境预热

## v0.3 — Workflow 编译器增强

**目标：让编译器能自动发现"同一类任务"，最大化模板复用率。**

### 功能清单

- [ ] **语义聚类** — 自动识别相似工作流，合并为更通用的参数化模板
- [ ] **模板泛化** — 更强的参数检测：服务名、日期范围、阈值、正则表达式等
- [ ] **工作流组合引擎** — 多个小模板拼接成复杂流程，支持条件分支
- [ ] **模板版本管理** — 编译后的模板支持迭代优化，保留版本历史
- [ ] **模板市场** — 预编译的常用工作流模板库，开箱即用
- [ ] **社区贡献机制** — 用户可提交模板到公共仓库，可评分和评论
- [ ] **自动模板测试** — 模板编译后自动回归测试，确保参数化未破坏功能

### 应用场景

- 运维巡检：50+ 标准操作全部编译为模板，新人用自然语言即可触发
- 数据处理：数据清洗、ETL 流程编译后跨项目复用
- 代码生成：常见代码片段生成模板，避免每次都调 LLM

## v0.4 — Agent 工具记忆系统

**目标：Agent 的所有工具调用历史自动索引，形成长期记忆，越用越聪明。**

### 功能清单

- [ ] **工具调用自动索引** — 每次工具调用的输入/输出/结果自动向量化存储
- [ ] **工具链推荐引擎** — 基于历史记录，自动推荐"这类问题通常需要调哪几个工具"
- [ ] **自我优化回路** — 统计工具调用成功率、平均耗时，自动推荐更优替代方案
- [ ] **异常模式检测** — 识别高频失败的工具调用模式，主动告警
- [ ] **跨 Agent 经验共享** — 多个 Agent 的工具调用经验汇聚到共享向量库
- [ ] **RAG 增强** — 历史工具执行结果作为 LLM 的上下文，提高推理质量
- [ ] **智能回退策略** — 工具调用失败时自动查询历史中相似场景的成功经验

### 应用场景

- OpenClaw 运行三个月后，80% 的日常操作绕过 LLM 直接执行
- 团队知识沉淀：所有 agent 的操作经验形成可搜索的知识库
- 新人上手：推荐引擎自动给出"这类问题通常怎么处理"的最佳实践
